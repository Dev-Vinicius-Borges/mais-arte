# Padrão de Pull Requests (PR)

**Versão:** 1.0.0
**Data:** 2026-07-19
**Status:** Ativo
**Responsável:** Vinícius Borges

---

## 1. Objetivo

Definir regras de governança para uso de Pull Requests (PR) no projeto, garantindo
rastreabilidade, priorização e execução previsível.

> O conteúdo descritivo do PR é definido pelo template oficial em
> `.github/pull_request_template.md`.

---

## 2. Escopo

Aplica-se a toda demanda registrada no repositório que envolva alterações de código, documentação ou configuração.

---

## 3. Fonte oficial de estrutura

A estrutura textual de cada PR deve seguir o template já existente em:

- `.github/pull_request_template.md`

---

## 4. Regras obrigatórias de criação

Toda PR deve:

- usar o template oficial;
- ter título objetivo e específico;
- relacionar a issue correspondente (quando aplicável);
- descrever claramente o que foi feito, como validar e evidências de validação;
- registar impactos técnicos, incluindo banco de dados, API/contratos, front-end e segurança/permissões;
- documentar atualizações relevantes, como README, documentação de processo, regras de negócio, arquitetura, modelo de dados e ADR (quando decisão arquitetural);
- declarar o uso de IA, se aplicável, conforme política do projeto.

## 5. Conformidade com o padrão de PR

Uma PR está conforme quando:

- [ ] Segue o template oficial;
- [ ] Título é objetivo e específico;
- [ ] Relaciona a issue correspondente (quando aplicável);
- [ ] Descreve claramente o que foi feito, como validar e evidências de validação;
- [ ] Registra impactos técnicos, incluindo banco de dados, API/contratos, front-end e segurança/permissões;
- [ ] Documenta atualizações relevantes, como README, documentação de processo, regras de negócio, arquitetura, modelo de dados e ADR (quando decisão arquitetural);
- [ ] Declara o uso de IA, se aplicável, conforme política do projeto.
